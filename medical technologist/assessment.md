# Performance Evaluator Skill

สร้าง interactive HTML artifact สำหรับประเมินผลงานพนักงานด้วย Claude API โดยรับแบบฟอร์มเกณฑ์ประเมินและรายงานผลงาน แล้วให้ AI วิเคราะห์และให้คะแนนพร้อมเหตุผล

## เมื่อ skill นี้ถูกเรียกใช้

ให้สร้าง React artifact หรือ HTML artifact ที่มีส่วนประกอบครบตามนี้ — ดู [โครงสร้าง artifact](#โครงสร้าง-artifact) ด้านล่าง

ถามผู้ใช้ก่อนสร้างถ้ายังไม่ทราบ:
1. **หัวข้อประเมิน** — fixed (กำหนดไว้แล้ว) หรือ dynamic (ผู้ใช้ปรับเองทุกครั้ง)?
2. **รูปแบบรายงาน** — พิมพ์ข้อความ / อัปโหลดไฟล์ / ทั้งสองแบบ?
3. **ผลลัพธ์ที่ต้องการ** — export PDF/Word ด้วยไหม นอกจากแสดงในหน้าเว็บ?

ถ้าผู้ใช้ไม่ระบุ ให้ใช้ค่า default: หัวข้อ fixed 7 หัวข้อ, รับทั้งพิมพ์และอัปโหลด, แสดงผลในหน้าเว็บ

---

## โครงสร้าง artifact

### 1. ส่วนข้อมูลพนักงาน
Input fields: ชื่อ-นามสกุล, ตำแหน่ง, แผนก, รอบการประเมิน

### 2. ส่วนเกณฑ์การประเมิน
แต่ละแถวมี:
- **ชื่อหัวข้อ** (text input)
- **น้ำหนักคะแนน** (number input) — ผลรวมต้องเท่ากับ 100
- **เกณฑ์การให้คะแนน** (textarea) — อธิบายว่าคะแนนสูง/กลาง/ต่ำหมายถึงอะไร

แสดง validator แบบ real-time: ถ้ารวมไม่ถึง 100 = แสดงเตือน, ถ้าเกิน 100 = แสดงสีแดง

**หัวข้อ default (7 หัวข้อ รวม 100 คะแนน):**
| หัวข้อ | น้ำหนัก |
|--------|---------|
| คุณภาพของงาน | 20 |
| ปริมาณและความตรงเวลา | 15 |
| การทำงานเป็นทีม | 15 |
| ความคิดสร้างสรรค์และการพัฒนา | 15 |
| ความรับผิดชอบและวินัย | 15 |
| การสื่อสารและรายงานผล | 10 |
| ทักษะและความเชี่ยวชาญ | 10 |

### 3. ส่วนรายงานผลงาน
- textarea สำหรับวางข้อความ
- upload zone สำหรับไฟล์ .txt (FileReader API) — PDF ต้องแปลงข้อความมาก่อน
- ถ้าผู้ใช้ต้องการรองรับ PDF จริงๆ ใช้ `pdf.js` จาก cdnjs

### 4. ปุ่มประเมิน
ตรวจสอบก่อน call API:
- [ ] มีรายงานผลงาน
- [ ] น้ำหนักรวม = 100
แสดง error inline ถ้าไม่ผ่าน

## การเรียก Claude API

```javascript
const systemPrompt = `คุณเป็นผู้ช่วยประเมินผลการปฏิบัติงานพนักงาน มีความเป็นกลาง ให้ข้อมูลที่มีประโยชน์ และซื่อสัตย์เมื่อข้อมูลไม่เพียงพอ

กฎสำคัญ:
1. ให้คะแนนแต่ละหัวข้อตามเกณฑ์ที่กำหนด โดยอ้างอิงหลักฐานจากรายงานเท่านั้น
2. หากข้อมูลในรายงานไม่เพียงพอสำหรับหัวข้อใด ให้ระบุ unsure: true และอธิบายว่าขาดข้อมูลอะไร
3. ห้ามคาดเดาหรือสมมติข้อมูลที่ไม่มีในรายงาน
4. ตอบเป็นภาษาไทยเท่านั้น
5. ตอบกลับเป็น JSON เท่านั้น ไม่มี preamble ไม่มี markdown code block`;

// โครงสร้าง JSON ที่คาดหวัง:
const expectedJSON = {
  scores: [
    {
      name: "ชื่อหัวข้อ",
      score: 17,          // null ถ้า unsure: true
      maxScore: 20,
      unsure: false,
      reason: "เหตุผลการให้คะแนน",
      evidence: "ข้อความอ้างอิงจากรายงาน"
    }
  ],
  totalScore: 82,
  summary: "สรุปภาพรวม 2-3 ประโยค",
  strengths: "จุดแข็งที่เด่นชัด",
  improvements: "ข้อเสนอแนะการพัฒนา"
};
```

**model:** ใช้ `claude-sonnet-4-20250514` เสมอ  
**max_tokens:** 1000  
**error handling:** ครอบ JSON.parse ด้วย try/catch, strip markdown backticks ก่อน parse

## การแสดงผล

### ส่วนหัวผลลัพธ์
- ชื่อพนักงาน + รอบประเมิน
- คะแนนรวมขนาดใหญ่ พร้อมสีตาม threshold:
  - ≥ 85% → success (เขียว)
  - ≥ 65% → neutral (ดำ/ขาว)
  - ≥ 50% → warning (เหลือง)
  - < 50% → danger (แดง)
- หมายเหตุ `* มีบางหัวข้อที่ยังไม่แน่ใจ` ถ้า unsure มีอยู่

### แต่ละหัวข้อประเมิน
```
[ชื่อหัวข้อ]                    [คะแนน / เต็ม]
[badge ⚠️ ไม่แน่ใจ ถ้า unsure]
[เหตุผล]
[หลักฐานอ้างอิง — ตัวเอียง สีอ่อน]
```

### ส่วนสรุป
- สรุปภาพรวม
- จุดแข็ง
- ข้อเสนอแนะพัฒนา

## ข้อควรระวัง

- **ไม่รองรับ PDF โดยตรง** — FileReader อ่านได้แค่ text ให้แจ้งผู้ใช้ว่าต้อง copy ข้อความมาวางแทน หรือใช้ไฟล์ .txt; ถ้าผู้ใช้ต้องการ PDF จริงๆ ให้ load `pdf.js` จาก cdnjs
- **ไม่ใช้ localStorage** — เก็บ state ใน JS variables หรือ React state เท่านั้น
- **น้ำหนักรวม ≠ 100** — ห้าม call API, แสดง error แทน
- **ข้อมูลพนักงานว่าง** — ใส่ค่า default "ไม่ระบุ" แทนการบังคับกรอก

## การ customize ที่พบบ่อย

| ความต้องการ | วิธีปรับ |
|-------------|---------|
| เพิ่ม/ลดหัวข้อ | แก้ `DEFAULT_CRITERIA` array |
| หัวข้อ dynamic (ผู้ใช้เพิ่มเองได้) | เพิ่มปุ่ม "เพิ่มหัวข้อ" + ปุ่มลบแต่ละแถว |
| Export PDF | เพิ่ม `window.print()` + print stylesheet |
| บันทึกประวัติ | ใช้ persistent storage API ถ้ามี |
| ภาษาอังกฤษ | แปล labels + system prompt |
