# Slide-to-Markdown Converter

แปลง PDF สไลด์เรียนเป็นไฟล์ Markdown ที่อ่านง่าย จัด format สวยงาม และเนื้อหาครบถ้วนทุกหน้า รวมถึงหน้าที่เป็นรูปภาพ

---

## หลักการสำคัญ

1. **เนื้อหาต้องครบ 100%** — ห้ามตัด ย่อ หรือละเว้นเนื้อหาจากสไลด์ใดๆ ทั้งสิ้น แม้แต่ bullet เดียว ตัวเลขเดียว หรือ footnote เล็กๆ ก็ต้องอยู่ครบ

2. **ภาษาคงเดิมตามสไลด์** — ถ้าสไลด์เป็นไทย ให้เป็นไทย ถ้าเป็นอังกฤษให้เป็นอังกฤษ ถ้าปนกันให้คงปนไว้ตามต้นฉบับ ห้ามแปลหรือเปลี่ยนภาษาเด็ดขาด

3. **จัด format เพื่อให้อ่านง่าย** — Claude มีอิสระในการเลือกใช้ heading, bullet, table, bold, italic ให้สวยงามและเข้าใจง่ายกว่าสไลด์ต้นฉบับ แต่ต้องไม่ทำให้เนื้อหาหายไป

4. **หน้ารูปภาพต้องอธิบาย** — ห้ามข้ามหน้าที่เป็นรูปภาพ ให้อธิบายสิ่งที่เห็นในภาพ และสรุปสิ่งที่ผู้เรียนควรรู้จากภาพนั้น (ดูรายละเอียดในหัวข้อ "การจัดการหน้ารูปภาพ")

5. **Checklist บนและล่างไฟล์** — ทุกครั้งต้องมี checklist แสดงสถานะทุกหน้าทั้งที่ต้นและท้ายไฟล์

6. **ตรวจสอบกับข้อมูลอินเทอร์เน็ต** — ทุก topic ให้ค้นหาข้อมูลเพิ่มเติมจากอินเทอร์เน็ต ถ้าพบข้อมูลที่ไม่ตรงกับสไลด์ให้แสดง ‼️ พร้อม reference ที่น่าเชื่อถือ (ดูรายละเอียดในหัวข้อ "การตรวจสอบความถูกต้อง")

7. **ระบุเลขหน้าของแต่ละหัวข้อ** — ทุก heading หลักต้องระบุช่วงเลขหน้าในสไลด์ต้นฉบับ

8. **สิ่งที่ต้องรู้เป็นภาษาไทย** — ทุก topic ต้องมี section "สิ่งที่ต้องรู้" เป็นภาษาไทย

9. **ใส่เหตุผลและกลไกสำหรับเนื้อหาที่ให้มาเฉยๆ** — เนื้อหาที่ระบุข้อเท็จจริงโดยไม่มีบริบท ให้อธิบายว่าใช้ทางการแพทย์อย่างไร หรืออธิบายกลไกการเกิด

10. **แหล่งที่มาต้องเป็นแหล่งความรู้ทางการแพทย์** — ห้ามอ้างอิงสำนักพิมพ์ที่ตีพิมพ์ ต้องเป็นแหล่งข้อมูลความรู้ทางการแพทย์ที่น่าเชื่อถือ (ดูรายการใน "แหล่งข้อมูลที่ยอมรับได้")

## โครงสร้าง Output (เรียงตามลำดับในไฟล์)

### 1. Checklist ต้นไฟล์

วางไว้เป็นส่วนแรกสุดของไฟล์ ก่อนเนื้อหาใดๆ

```markdown
# 📋 Topic Checklist — สิ่งที่ต้องรู้ทั้งหมด

| หน้า | หัวข้อสไลด์ | สถานะ |
|------|-------------|--------|
| 1    | ชื่อวิชา / Title slide | ⬜ |
| 2    | Objectives | ⬜ |
| 3    | ... | ⬜ |
```

---

### 2. เนื้อหาหลัก — โครงสร้างต่อ Section

ทุก section (หัวข้อหลักจากสไลด์) ต้องมีองค์ประกอบครบดังนี้:

```markdown
## [ชื่อหัวข้อ] — หน้า X–Y

> 📍 **ตำแหน่งในเอกสาร:** หน้า X–Y

### เนื้อหาจากสไลด์
[เนื้อหาครบถ้วนตามสไลด์ต้นฉบับ ไม่ตัดไม่ย่อ]

### 🧠 กลไกและเหตุผลทางการแพทย์
[อธิบาย "ทำไม" ของเนื้อหาในสไลด์ — กลไกการเกิด (pathophysiology/mechanism) และ/หรือ
การนำไปใช้ทางคลินิก (clinical application) ห้ามแค่ list ข้อมูลโดยไม่มี context]

### 📌 สิ่งที่ต้องรู้ (ภาษาไทย)
- สิ่งสำคัญที่ผู้เรียนต้องจำและเข้าใจจาก topic นี้ เขียนเป็นภาษาไทยทั้งหมด
- ระบุเป็น bullet point ชัดเจน ครอบคลุม key concepts, clinical pearls, และ red flags

### ‼️ ข้อมูลที่อาจไม่ตรงกับสไลด์
[ถ้าไม่พบความขัดแย้ง ให้เขียน: ✅ ข้อมูลสอดคล้องกับแหล่งอ้างอิง]
[ถ้าพบความขัดแย้ง ให้เขียน:
‼️ [ระบุประเด็นที่แตกต่าง]
- **ในสไลด์:** ...
- **ตามแหล่งอ้างอิง:** ...
- **แหล่งอ้างอิง:** [ชื่อแหล่ง + URL หรือ DOI]
]
```

### 3. Checklist ท้ายไฟล์

```markdown
# 📋 Progress Checklist

| หน้า | หัวข้อสไลด์ | สถานะ |
|------|-------------|--------|
| 1    | ชื่อวิชา / Title slide | ✅ |
| 6    | Severity graph | 🖼️ อธิบายภาพแล้ว |

**แปลงแล้ว: 30/30 หน้า (รวม 2 หน้าที่เป็นรูปภาพ)**
```

## การตรวจสอบความถูกต้อง (ข้อ 6)

ทุก topic ให้ใช้ web search ค้นหาข้อมูลเพื่อตรวจสอบ โดยเฉพาะ:
- ตัวเลขทางคลินิก (ค่า cutoff, dosage, reference range)
- คำแนะนำการรักษา (guidelines)
- กลไกที่อธิบายไว้
- การจำแนกประเภท/classification

**กฎการแสดงผล:**
- ✅ ข้อมูลสอดคล้อง → เขียนสั้นๆ ว่า "ข้อมูลสอดคล้องกับแหล่งอ้างอิง"
- ‼️ พบความขัดแย้ง → แสดงทั้งสองฝั่งพร้อม reference

**ตัวอย่างการแสดงผลเมื่อพบความขัดแย้ง:**
```
‼️ เกณฑ์การวินิจฉัยในสไลด์อาจล้าสมัย
- **ในสไลด์:** HbA1c ≥ 6.5% ใช้ได้เฉพาะวิธี NGSP
- **ตามแหล่งอ้างอิง:** ADA 2024 ยอมรับทั้ง NGSP และ IFCC โดยต้องระบุหน่วยด้วย
- **แหล่งอ้างอิง:** American Diabetes Association Standards of Care 2024 — https://diabetesjournals.org/care
```

## แหล่งข้อมูลที่ยอมรับได้ (ข้อ 10)

ใช้เฉพาะแหล่งที่เป็นความรู้ทางการแพทย์โดยตรง **ห้ามอ้างสำนักพิมพ์** (เช่น ห้ามอ้าง "Elsevier", "Wiley", "Lippincott" เป็นแหล่งอ้างอิง)

**แหล่งที่ยอมรับได้:**
| ประเภท | ตัวอย่าง |
|--------|----------|
| Clinical guidelines | ADA, AHA, ACC, WHO, NICE, ACOG, IDSA |
| Evidence databases | UpToDate, PubMed/MEDLINE, Cochrane Library |
| Reference tools | StatPearls (NCBI Bookshelf), Medscape |
| Pharmacology | FDA Drug Label, BNF, Micromedex |
| Pathology/Lab | CLSI, CAP, RCPA |
| Thai guidelines | แนวทางเวชปฏิบัติของราชวิทยาลัยแพทย์ไทย, กรมการแพทย์ |

**รูปแบบการอ้างอิงที่ถูกต้อง:**
- `WHO Guidelines on Malaria Treatment (2023) — https://www.who.int/...`
- `UpToDate: Management of type 2 diabetes mellitus — uptodate.com`
- `AHA/ACC 2023 Hypertension Guidelines — doi:10.1161/...`
- `StatPearls: Acute Pancreatitis — NCBI Bookshelf`

**ห้ามอ้างอิงในลักษณะนี้:**
- ❌ "Robbins & Cotran Pathologic Basis of Disease (Elsevier)"
- ❌ "Harrison's Principles (McGraw-Hill)"
- ❌ "สำนักพิมพ์..."

## การจัด Format

### Heading Hierarchy

| ระดับ | ใช้สำหรับ |
|-------|-----------|
| `#`   | ชื่อวิชา — ใช้ครั้งเดียวตอนต้นไฟล์ |
| `##`  | หัวข้อหลักของแต่ละสไลด์ (พร้อมเลขหน้า) |
| `###` | Sub-sections: เนื้อหาจากสไลด์ / กลไก / สิ่งที่ต้องรู้ / ข้อมูลที่ต่างกัน |
| `####`| หัวข้อย่อยลึกกว่านั้น (ใช้เมื่อจำเป็น) |

### การเลือกใช้ Element

- **Bullet points** (`-`) — รายการทั่วไปที่ไม่มีลำดับ
- **Numbered list** (`1.`) — รายการที่มีลำดับ, ขั้นตอนการรักษา, กลไกที่เกิดตามลำดับ
- **ตาราง Markdown** — ข้อมูล tabular หรือ comparison (สร้างตารางแทน bullet เสมอเมื่อข้อมูลมีมากกว่า 1 มิติ)
- **`Bold`** — keyword, ชื่อโรค/ยาที่ต้องจำ
- **`*Italic*`** — source citation, คำอธิบายรูป
- **`` `code` ``** — สูตรเคมี, receptor names, ชื่อยีน เช่น `GABA-A`, `CYP3A4`
- **สัญลักษณ์** — คงเดิม: `→`, `↑`, `↓`, `∝`, `µ`, `~`, `±`, `≥`, `≤`
- **`---`** — คั่นระหว่างแต่ละ section เสมอ

### Case Study Slides

```markdown
## Case Study: [ชื่อเคส] — หน้า X

### Chief Complaint (CC)
### History of Present Illness (HPI)
### Physical Examination (PE)
### Laboratory / Investigations
### Discussion / Key Points

### 🧠 กลไกและเหตุผลทางการแพทย์
### 📌 สิ่งที่ต้องรู้ (ภาษาไทย)
### ‼️ ข้อมูลที่อาจไม่ตรงกับสไลด์
```

## การจัดการหน้ารูปภาพ

**ห้ามข้ามหน้าที่มีรูปภาพ** ให้ทำตามนี้ทุกครั้ง:

### หน้าที่มีรูปภาพ + มีข้อความกำกับ
แปลงข้อความทุกตัวตามปกติ แล้วเพิ่มบล็อกอธิบายภาพต่อท้าย

### หน้าที่เป็นรูปภาพล้วน

```markdown
## [หัวข้อสไลด์ หรือ "ภาพประกอบ — หน้า X"] — หน้า X

> 🖼️ **คำอธิบายภาพ:** [อธิบายว่าภาพแสดงอะไร เช่น กราฟ, แผนภาพ, flow chart ฯลฯ
> บอกแกน X Y, กลุ่มที่เปรียบเทียบ, trend ที่สำคัญ]

> 📚 **สิ่งที่ควรรู้จากภาพนี้:** [สรุปสาระสำคัญเชิงคลินิก]

### 🧠 กลไกและเหตุผลทางการแพทย์
[อธิบายว่าภาพนี้สื่อถึงกลไกหรือ clinical concept อะไร]

### 📌 สิ่งที่ต้องรู้ (ภาษาไทย)
[bullet points ภาษาไทย]

### ‼️ ข้อมูลที่อาจไม่ตรงกับสไลด์
[ตรวจสอบตามปกติ]
```

ใน checklist ให้ใช้สถานะ 🖼️ อธิบายภาพแล้ว สำหรับหน้าประเภทนี้

## ขั้นตอนการทำงาน

1. **อ่าน PDF และนับหน้าทั้งหมด** — ระบุจำนวนหน้าและหัวข้อคร่าวๆ ของทุกหน้า

2. **สร้าง Checklist ต้นไฟล์** — ระบุทุกหน้าพร้อมหัวข้อ สถานะเริ่มต้นเป็น ⬜ วางก่อนเนื้อหา

3. **แปลงเนื้อหาทีละหน้า** — ทำตามลำดับหน้า ไม่ข้าม ไม่สลับ:
   - แปลงเนื้อหาครบจากสไลด์
   - ระบุเลขหน้าใน heading
   - เพิ่ม section กลไก/เหตุผลทางการแพทย์
   - เพิ่ม section สิ่งที่ต้องรู้ (ภาษาไทย)
   - ค้นหาข้อมูลเพิ่มเติมและแสดง ‼️ ถ้าพบความขัดแย้ง

4. **ใช้ `---` คั่นทุก section**

5. **สรุป Checklist ท้ายไฟล์** — Update สถานะทุกหน้า พร้อมสรุปจำนวนรวม

6. **ส่งไฟล์** ด้วย present_files

## ข้อห้าม

- ❌ ห้ามสรุปหรือย่อเนื้อหาจากสไลด์
- ❌ ห้ามแปลภาษา ภาษาใดมาภาษานั้นไป
- ❌ ห้ามข้ามหน้าใดๆ โดยไม่มีใน checklist
- ❌ ห้ามข้ามหน้ารูปภาพโดยไม่อธิบาย
- ❌ ห้ามลืมทำ checklist ทั้งต้นและท้ายไฟล์
- ❌ ห้ามอ้างอิงสำนักพิมพ์เป็นแหล่งข้อมูล
- ❌ ห้ามให้เนื้อหาเพียงอย่างเดียวโดยไม่มีกลไกหรือเหตุผลทางการแพทย์
- ❌ ห้ามเขียน section "สิ่งที่ต้องรู้" เป็นภาษาอื่นนอกจากภาษาไทย
- ❌ ห้ามลืมระบุเลขหน้าใน heading หลักทุกอัน
