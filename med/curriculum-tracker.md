# Curriculum Tracker — Cross-Reference & Map Updater

วิเคราะห์โน้ตบทเรียนใหม่เทียบกับสิ่งที่เรียนไปแล้ว ระบุความซ้ำซ้อน สกัดความรู้ใหม่ และอัปเดตดัชนีสะสม

---

## 📥 Input ที่ต้องการ

| ไฟล์ | คำอธิบาย | ถ้าไม่มี |
|------|----------|----------|
| `notes-synthesized.md` | โน้ตบทเรียนใหม่ที่สังเคราะห์จากสไลด์+เสียงอาจารย์แล้ว | **จำเป็น** — หยุดและขอจากผู้ใช้ |
| `Curriculum_Map.md` | ดัชนีความรู้สะสมจากทุกบทเรียนที่ผ่านมา | ถือว่าเป็นบทเรียนแรก → ข้าม cross-reference, สร้าง Map ใหม่เลย |

---

## ⚙️ ขั้นตอนการประมวลผล

### Step 0 — อ่านและทำความเข้าใจ Input
1. อ่าน `notes-synthesized.md` ทั้งหมด บันทึกรายการหัวข้อและ key concepts ทุกอัน
2. อ่าน `Curriculum_Map.md` (ถ้ามี) เพื่อโหลดรายการหัวข้อที่เคยเรียนมาแล้ว
3. ถ้า Map ว่างเปล่า → ข้ามไป Step 2 ได้เลย (ทุกอย่างเป็น Type C)

---

### Step 1 — ตรวจจับความซ้ำซ้อน (Semantic Overlap Detection)

สำหรับแต่ละ **section/หัวข้อ** ใน notes ใหม่ ให้จัดเป็น 1 ใน 3 ประเภท:

| ประเภท | สัญลักษณ์ | เกณฑ์ |
|--------|-----------|-------|
| **Type A** | 🔴 Full Overlap | ซ้ำเกือบ 100% — ทฤษฎีเดิม ไม่มีข้อมูลใหม่เพิ่มขึ้นเลย |
| **Type B** | 🟡 Partial Overlap | หัวข้อเดิม แต่มีมุมมองใหม่ / clinical case ใหม่ / คำอธิบายต่างออกไป |
| **Type C** | 🟢 New Topic | ไม่เคยปรากฏในบทเรียนใดมาก่อน |

> **หมายเหตุ:** ใช้ความหมาย (semantic similarity) ไม่ใช่แค่ชื่อหัวข้อ — หัวข้อต่างชื่อแต่สอนเรื่องเดียวกัน = Type A หรือ B

---

### Step 2 — จัดการเนื้อหาตามประเภท

#### 🔴 Type A: Full Overlap — ยุบย่อ + ชี้เป้าบทเรียนเดิม

ตัดรายละเอียดลึกออก เหลือแค่ callout สั้น:

```markdown
> 📖 **เคยเรียนแล้วใน:** **[Lecture XX: ชื่อบทเรียน]** (หน้า Y)
> **สรุปสั้น:** [1–2 ประโยค กันลืม]
> *(ทบทวนรายละเอียดได้ที่โน้ตบทเรียนเดิม)*
```

**เมื่อไรควรเหลือเนื้อหาต้นฉบับไว้บ้าง:** ถ้าหัวข้อมีสูตร/ตาราง/diagram สำคัญที่ต้องอ้างอิงเร็ว ให้คงไว้ใต้ callout พร้อม label `[คงไว้เพื่ออ้างอิงเร็ว]`

---

#### 🟡 Type B: Partial Overlap — คงบริบทเดิม + เน้นสิ่งใหม่

คง section เดิมไว้เป็น "บริบท" แต่เพิ่ม callout ใหม่:

```markdown
> 💡 **ความรู้เพิ่มเติมจากบทเรียนนี้ (New Nuances):**
> *หัวข้อนี้เคยเรียนแล้วใน [Lecture XX] แต่บทเรียนนี้เพิ่มเติม:*
> - [ข้อมูลใหม่ / clinical pearl / มุมมองต่าง ๆ ที่ไม่เคยมีในบทเรียนก่อน]
> - [ถ้ามีหลายข้อ ให้แยก bullet]
```

**สิ่งที่ถือเป็น "ข้อมูลใหม่":** clinical case ใหม่, exception ที่ไม่เคยพูดถึง, คำศัพท์เฉพาะใหม่, ข้อมูลที่อาจารย์เน้นว่า "ออกสอบ", ตัวเลข/เกณฑ์ที่ต่างจากเดิม

---

#### 🟢 Type C: New Topic — คงไว้ครบ 100%

**ห้ามตัดทอนใด ๆ ทั้งสิ้น** คงโครงสร้างและเนื้อหาตามต้นฉบับ เพิ่มเพียง label เล็กน้อยว่าเป็นหัวข้อใหม่ (optional):

```markdown
<!-- 🟢 New Topic — เพิ่มใน Curriculum Map แล้ว -->
```

---

### Step 3 — อัปเดต Curriculum Map

หลังประมวลผลโน้ตครบแล้ว ให้เพิ่มรายการต่อไปนี้ลงใน `Curriculum_Map.md`:

- ✅ **Type C** ทุกหัวข้อ — เพิ่มแน่นอน
- ✅ **Type B** ที่มี new nuance สำคัญ — เพิ่มพร้อม note ว่ามีอัปเดตจากบทเรียนนี้
- ❌ **Type A** — ไม่ต้องเพิ่ม (มีอยู่แล้ว)

**รูปแบบ entry ใน Curriculum Map:**

```markdown
## [Lecture XX: ชื่อบทเรียน] — [วันที่ / ลำดับบทเรียน]

### หัวข้อสำคัญ
- **[ชื่อหัวข้อ]** — [คำอธิบาย 1 บรรทัด] | Type: C / B-updated
- **[ชื่อหัวข้อ]** — [คำอธิบาย 1 บรรทัด] | Type: C
```

---

## 📄 Output ที่ต้องส่งมอบ (เสมอ 2 ไฟล์)

### ไฟล์ที่ 1: `notes-optimized.md`
โน้ตที่จัดแล้ว — เนื้อหาซ้ำยุบแล้ว, เนื้อหาใหม่ครบ, callout ชัดเจน

### ไฟล์ที่ 2: `Curriculum_Map_updated.md`
Map เวอร์ชันล่าสุด — ใช้เป็น input สำหรับบทเรียนถัดไป

---

## 🚨 Edge Cases & การจัดการ

| สถานการณ์ | วิธีจัดการ |
|-----------|-----------|
| `Curriculum_Map.md` ว่างเปล่า / ไม่มีไฟล์ | ทุก section เป็น Type C → สร้าง Map ใหม่จากบทเรียนนี้เลย |
| หัวข้อในโน้ตอ้างถึงบทเรียนก่อนโดยตรง | ถือเป็น Type B อย่างน้อย — ตรวจหา new nuance |
| section เดียวมีทั้ง Type B และ C ปนกัน | แยก sub-section: ยุบ/ไฮไลต์ส่วน B, คงส่วน C ครบ |
| ไม่แน่ใจว่าซ้ำหรือเปล่า | ถือเป็น Type B (conservative) — ดีกว่าตัดข้อมูลที่อาจสำคัญทิ้ง |
| บทเรียนนี้แก้ไขข้อมูลเดิมที่ผิด | ระบุชัดใน callout ว่า "ข้อมูลนี้แก้ไขจาก [Lecture XX]" |
