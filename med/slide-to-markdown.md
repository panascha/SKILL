# Slide-to-Markdown Converter

แปลง PDF สไลด์เรียนเป็นไฟล์ Markdown ที่อ่านง่าย จัด format สวยงาม และเนื้อหาครบถ้วนทุกหน้า รวมถึงหน้าที่เป็นรูปภาพ

---

## หลักการสำคัญ

1. **เนื้อหาต้องครบ 100%** — ห้ามตัด ย่อ หรือละเว้นเนื้อหาจากสไลด์ใดๆ ทั้งสิ้น แม้แต่ bullet เดียว ตัวเลขเดียว หรือ footnote เล็กๆ ก็ต้องอยู่ครบ

2. **ภาษาคงเดิมตามสไลด์** — ถ้าสไลด์เป็นไทย ให้เป็นไทย ถ้าเป็นอังกฤษให้เป็นอังกฤษ ถ้าปนกันให้คงปนไว้ตามต้นฉบับ ห้ามแปลหรือเปลี่ยนภาษาเด็ดขาด

3. **จัด format เพื่อให้อ่านง่าย** — Claude มีอิสระในการเลือกใช้ heading, bullet, table, bold, italic ให้สวยงามและเข้าใจง่ายกว่าสไลด์ต้นฉบับ แต่ต้องไม่ทำให้เนื้อหาหายไป

4. **หน้ารูปภาพต้องอธิบาย** — ห้ามข้ามหน้าที่เป็นรูปภาพ ให้อธิบายสิ่งที่เห็นในภาพ และสรุปสิ่งที่ผู้เรียนควรรู้จากภาพนั้น (ดูรายละเอียดในหัวข้อ "การจัดการหน้ารูปภาพ")

5. **Checklist บนและล่างไฟล์** — ทุกครั้งต้องมี checklist แสดงสถานะทุกหน้าทั้งที่ต้นและท้ายไฟล์

---

## โครงสร้าง Output (เรียงตามลำดับในไฟล์)

### 1. Checklist ต้นไฟล์

วางไว้เป็นส่วนแรกสุดของไฟล์ ก่อนเนื้อหาใดๆ เพื่อให้ผู้อ่านรู้ว่าสไลด์ทั้งหมดมีกี่หน้า และครอบคลุมอะไรบ้าง

```markdown
# 📋 Topic Checklist — สิ่งที่ต้องรู้ทั้งหมด

| หน้า | หัวข้อสไลด์ | สถานะ |
|------|-------------|--------|
| 1  | ชื่อวิชา / Title slide | ⬜ |
| 2  | Objectives | ⬜ |
| 3  | ... | ⬜ |

---
```

### 2. เนื้อหาหลัก

```markdown
# [ชื่อวิชา]
> ผู้สอน: ... | ภาควิชา: ... | อีเมล: ...

---

## [หัวข้อสไลด์หน้าที่ 2]

เนื้อหา...

---

## [หัวข้อสไลด์หน้าที่ 3]

เนื้อหา...

---
```

### 3. Checklist ท้ายไฟล์

วางซ้ำท้ายไฟล์อีกครั้งพร้อม update สถานะทุกหน้าเป็น ✅ และสรุปจำนวน

```markdown
# 📋 Progress Checklist

| หน้า | หัวข้อสไลด์ | สถานะ |
|------|-------------|--------|
| 1  | ชื่อวิชา / Title slide | ✅ |
| 2  | Objectives | ✅ |
| 6  | Severity graph | 🖼️ อธิบายภาพแล้ว |
| 30 | Management | ✅ |

**แปลงแล้ว: 30/30 หน้า (รวม 2 หน้าที่เป็นรูปภาพ)**
```

## การจัด Format

### Heading Hierarchy

| ระดับ | ใช้สำหรับ | ตัวอย่าง |
|-------|-----------|----------|
| `#`   | ชื่อวิชา — ใช้ครั้งเดียวตอนต้นไฟล์เท่านั้น | `# Pharmacology 301` |
| `##`  | หัวข้อหลักของแต่ละสไลด์ | `## Mechanisms of Action` |
| `###` | หัวข้อย่อยภายในสไลด์เดียวกัน | `### Beta-blocker Subtypes` |
| `####`| หัวข้อย่อยลึกกว่านั้น (ใช้เมื่อจำเป็นจริงๆ) | `#### Selective vs Non-selective` |

### การเลือกใช้ Element แต่ละประเภท

- **Bullet points** (`-`) — สำหรับรายการทั่วไปที่ไม่มีลำดับ เช่น อาการ, ข้อบ่งชี้, ผลข้างเคียง
- **Numbered list** (`1.`) — สำหรับรายการที่มีลำดับ เช่น ขั้นตอนการรักษา, กลไกที่เกิดตามลำดับ
- **ตาราง Markdown** — สำหรับข้อมูลที่เปรียบเทียบหรือ tabular เช่น อาการตาม blood level, drug comparison, withdrawal timeline ให้สร้างตารางแทน bullet เสมอเมื่อข้อมูลมีมากกว่า 1 มิติ
- **`Bold`** — สำหรับคำสำคัญ keyword หรือชื่อโรค/ยาที่ต้องจำ
- **`*Italic*`** — สำหรับ source citation, คำอธิบายรูป, คำศัพท์ภาษาต่างประเทศที่อ้างอิง
- **`` `code` ``** (backtick) — สำหรับสูตรเคมี, receptor names, ชื่อยีน เช่น `GABA-A`, `µ-opioid`, `CYP3A4`
- **สัญลักษณ์ทางวิทยาศาสตร์/การแพทย์** — คงเดิมทุกตัว: `→`, `↑`, `↓`, `∝`, `∆⁹`, `µ`, `~`, `±`, `≥`, `≤`
- **`---`** — คั่นระหว่างแต่ละสไลด์/section เสมอ

### Case Study Slides

สไลด์ที่เป็น case study หรือ clinical vignette ให้รวมข้อมูลไว้ใน section เดียวและจัด sub-heading ดังนี้:

```markdown
## Case Study: [ชื่อเคส หรือ Case 1]

### Chief Complaint (CC)
...

### History of Present Illness (HPI)
...

### Physical Examination (PE)
...

### Laboratory / Investigations
...

### Discussion / Key Points
...
```

## การจัดการหน้ารูปภาพ

**ห้ามข้ามหน้าที่มีรูปภาพ** ให้ทำตามนี้ทุกครั้ง:

### หน้าที่มีรูปภาพ + มีข้อความกำกับ
แปลงข้อความทุกตัวตามปกติ แล้วเพิ่มบล็อกอธิบายภาพต่อท้าย

### หน้าที่เป็นรูปภาพล้วน (ไม่มีข้อความเลย หรือมีแค่ caption สั้นๆ)
ให้สร้าง section ในลักษณะนี้:

```markdown
## [หัวข้อสไลด์ หรือ "ภาพประกอบ — หน้า X"]

> 🖼️ **คำอธิบายภาพ:** [อธิบายว่าภาพแสดงอะไร เช่น กราฟ, แผนภาพ, flow chart, ตาราง, microscopy image ฯลฯ บอกให้ละเอียดว่าแกน X Y คืออะไร, มีกี่กลุ่ม, trend เป็นอย่างไร]

> 📚 **สิ่งที่ควรรู้จากภาพนี้:** [สรุปสาระสำคัญที่ผู้เรียนควรเข้าใจหลังดูภาพนี้ เช่น ความสัมพันธ์ระหว่างตัวแปร, pattern ที่สำคัญ, ข้อสรุปเชิงคลินิก]
```

**ตัวอย่าง:**

```markdown
## Opioid Dose-Response Curve — หน้า 14

> 🖼️ **คำอธิบายภาพ:** กราฟ dose-response curve แสดงความสัมพันธ์ระหว่างขนาดยา (แกน X, log scale) และ % maximum effect (แกน Y) เปรียบเทียบระหว่าง morphine, codeine และ buprenorphine พบว่า morphine มี ceiling effect ต่ำกว่า buprenorphine ที่ dose สูง

> 📚 **สิ่งที่ควรรู้จากภาพนี้:** Buprenorphine เป็น partial agonist จึงมี ceiling effect — การเพิ่ม dose เกินจุดหนึ่งไม่ทำให้ฤทธิ์เพิ่มขึ้น แต่ก็ลดความเสี่ยง respiratory depression ในผู้ป่วย opioid use disorder ได้ดีกว่า full agonist
```

ใน checklist ให้ใช้สถานะ `🖼️ อธิบายภาพแล้ว` แทน ✅ สำหรับหน้าประเภทนี้

## ขั้นตอนการทำงาน

1. **อ่าน PDF และนับหน้าทั้งหมด** — ระบุจำนวนหน้าและหัวข้อคร่าวๆ ของทุกหน้า ก่อนเริ่มแปลง

2. **สร้าง Checklist ต้นไฟล์** — ระบุทุกหน้าพร้อมหัวข้อ สถานะเริ่มต้นเป็น ⬜ ทั้งหมด วางไว้ก่อนเนื้อหา

3. **แปลงเนื้อหาทีละหน้า** — ทำตามลำดับหน้า ไม่ข้าม ไม่สลับ:
   - หน้าปกติ → แปลงเนื้อหาครบ
   - หน้ารูปภาพ → อธิบายภาพ + สิ่งที่ควรรู้
   - หน้า case study → ใช้ format case study

4. **ใช้ `---` คั่นทุก section** — ทำให้ไฟล์อ่านง่ายและแบ่ง section ชัดเจน

5. **สรุป Checklist ท้ายไฟล์** — Update สถานะทุกหน้า พร้อมสรุปจำนวนรวม เช่น "แปลงแล้ว 28/30 หน้า (2 หน้าเป็นภาพ — อธิบายแล้ว)"

6. **ส่งไฟล์** ด้วย `present_files`

## ข้อห้าม

- ❌ ห้ามสรุปหรือย่อเนื้อหา แม้จะรู้สึกว่าซ้ำกัน
- ❌ ห้ามแปลภาษา ภาษาใดมาภาษานั้นไป
- ❌ ห้ามเพิ่มความคิดเห็น คำอธิบาย หรือ commentary ของ Claude เองนอกจากส่วน "อธิบายภาพ" ที่กำหนดไว้
- ❌ ห้ามข้ามหน้าใดๆ โดยไม่มีใน checklist
- ❌ ห้ามข้ามหน้ารูปภาพโดยไม่อธิบาย
- ❌ ห้ามลืมทำ checklist ทั้งต้นและท้ายไฟล์
- ❌ ห้ามเขียน checklist แค่ท้ายไฟล์ — ต้องมีทั้งสองด้าน
