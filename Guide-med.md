# 🗺️ Master Pipeline Overview: จากสไลด์ดิบสู่โน้ตตกผลึกชั้นเลิศ

การจัดเรียงลำดับ (Workflow Order) ที่ดีที่สุดเพื่อไม่ให้ข้อมูลตกหล่นและลดภาระสมอง มีดังนี้ครับ:

```
[1. แปลงไฟล์]      Lecture.pdf  ──► (slide-to-markdown) ──► notes-raw.md
                                                                │
[2. รวมเสียง]      notes-raw.md + transcript.txt ──► (lecture-synthesizer) ──► notes-synthesized.md
                                                                                  │
[3. เสริมความรู้]                                 notes-synthesized.md ──► (slide-enrich) ──► notes-enriched.md
                                                                                                  │
[4. จัดลำดับเนื้อหา]                                        notes-enriched.md ──► (note-resequencer) ──► notes-resequenced.md
                                                                                                              │
[5. ตรวจความซ้ำ]                       notes-resequenced.md + Curriculum_Map.md ──► (curriculum-tracker)
                                                                                          │
                                                                           ┌──────────────┴──────────────┐
                                                                           ▼                             ▼
                                                                  notes-optimized.md            Curriculum_Map_updated.md
                                                                           │
[6. ตกผลึก]                                             notes-optimized.md ──► (lecture-crystallizer) ──► notes-crystallized.md
```

---

## 🧭 รายละเอียดขั้นตอนการใช้งานทีละ Step

### 📌 ขั้นที่ 1: เตรียมโครงสร้างโน้ตจากสไลด์
- **Skill ที่ใช้:** `slide-to-markdown`
- **Input:** ไฟล์สไลด์อาจารย์อย่างเดียว (`Lecture.pdf`)
- **Output:** `notes-raw.md` (ไฟล์ Markdown ที่ถอดเนื้อหาและภาพสไลด์ออกมาครบ 100% พร้อมทำตารางและหัวข้อให้อ่านง่าย)
- **เหตุผล:** เป็นการสกัดเนื้อหาที่เป็นทางการจากสไลด์เพื่อใช้เป็น "กระดูกสันหลัง" ของบทเรียนก่อนนำไปผสมกับข้อมูลส่วนอื่น

### 📌 ขั้นที่ 2: ผสานคำอธิบายสดของอาจารย์ผู้สอน
- **Skill ที่ใช้:** `lecture-synthesizer`
- **Input:** `notes-raw.md` (จากขั้นที่ 1) + `transcript.txt` (เทปบันทึกเสียงพูดอาจารย์)
- **Output:** `notes-synthesized.md`
- **เหตุผล:** นำคำอธิบายสด ตัวอย่างเคส และจุดเน้นย้ำ ("Spoken Pearls") มาแปะแทรกไว้ใต้หัวข้อสไลด์ที่ถูกต้องทันที ทำให้เราได้โน้ตเนื้อหาที่ "ครบถ้วนตามจริงในห้องเรียน"

### 📌 ขั้นที่ 3: เติมความลึกและตรวจสอบความถูกต้องเชิงวิชาการ
- **Skill ที่ใช้:** `slide-enrich`
- **Input:** `notes-synthesized.md` (จากขั้นที่ 2)
- **Output:** `notes-enriched.md`
- **เหตุผล:**
  - ขั้นตอนนี้จะสแกนหาจุดที่เนื้อหาในสไลด์หรือคำพูดอาจารย์อาจจะสั้นเกินไป แล้วเขียนกลไกอธิบาย "ทำไม" (`🧠 กลไกและเหตุผลทางการแพทย์`) ร่วมกับการสรุปประเด็นหลักภาษาไทย (`📌 สิ่งที่ต้องรู้`)
  - ที่สำคัญคือมีการทำ **Web Search** เพื่อตรวจสอบข้อมูลตัวเลข แนวทางการรักษา (Guidelines) ว่าตรงกับฐานข้อมูลสากลล่าสุดหรือไม่ หากเจอจุดที่สไลด์ล้าสมัยจะแสดงสัญลักษณ์ `‼️` เตือนทันที

### 📌 ขั้นที่ 4: จัดลำดับเนื้อหาตามสายโซ่ความรู้ทางการแพทย์
- **Skill ที่ใช้:** `note-resequencer`
- **Input:** `notes-enriched.md` (จากขั้นที่ 3)
- **Output:** `notes-resequenced.md`
- **เหตุผล:**
  - เนื้อหาใน `notes-enriched.md` ยังเรียงตามลำดับสไลด์ของอาจารย์ ซึ่งมักไม่ใช่ลำดับที่เหมาะกับการเรียนรู้ด้วยตัวเอง
  - skill นี้จะ **cluster** และ **resequence** เนื้อหาทั้งหมดตาม Medical Learning Chain 7 ชั้น ได้แก่ Anchor → Foundation → Mechanism → Presentation → Workup → Management → Outcomes โดย **ไม่เพิ่มหรือลบเนื้อหาใดๆ**
  - ผลลัพธ์คือโน้ตที่ผู้เรียนสามารถไล่อ่านจากต้นจนจบได้อย่างมีเหตุมีผล เข้าใจ "ทำไม" ก่อนเสมอ แล้วจึงต่อยอดไปสู่ clinical features, investigation และ treatment ตามลำดับ

### 📌 ขั้นที่ 5: ตัดส่วนที่เคยเรียนไปแล้วออกข้ามคาบ
- **Skill ที่ใช้:** `curriculum-tracker`
- **Input:** `notes-resequenced.md` (จากขั้นที่ 4) + `Curriculum_Map.md` (ไฟล์แผนผังความรู้สะสมของวิชานี้)
- **Output:** ได้ 2 ไฟล์คือ:
  1. `notes-optimized.md` (โน้ตบทเรียนปัจจุบันที่ยุบส่วนซ้ำ และเหลือเฉพาะส่วนที่เป็นข้อมูลใหม่จริงๆ)
  2. `Curriculum_Map_updated.md` (แผนผังความรู้สะสมที่อัปเดตหัวข้อใหม่ของบทนี้เข้าไปแล้ว)
- **เหตุผล:** ป้องกันภาวะข้อมูลท้น (Cognitive Overload) โดยการเปรียบเทียบเชิงความหมายกับเลกเชอร์ก่อนๆ หัวข้อไหนที่เคยเรียนละเอียดแล้วจะถูกยุบเหลือเพียง Callout ชี้เป้ากลับไปที่เดิม ช่วยรักษาขนาดคลังข้อมูลให้กระชับ

### 📌 ขั้นที่ 6: ตกผลึกเป็นคู่มือเตรียมสอบและการนำไปใช้งาน
- **Skill ที่ใช้:** `lecture-crystallizer`
- **Input:** `notes-optimized.md` (จากขั้นที่ 5)
- **Output:** `notes-crystallized.md`
- **เหตุผล:** เป็นขั้นตอนสุดท้ายในการแปลงโน้ตเนื้อหาขนาดยาว ให้กลายเป็น **"สรุปแบบ High-Yield"** โดยจัดรูปแบบใหม่ออกมาเป็น 4 แกนสำคัญ (จุดเน้นสอบ, สายโซ่กลไกสั้นๆ, เทคนิคช่วยจำ/Mnemonic, แผนปฏิบัติหน้างานจริงบนวอร์ด) ซึ่งเหมาะอย่างยิ่งสำหรับการนำไปอ่านทวนเร็ว (Quick review) หรือทำ Flashcards

## 🗃️ การจัดระเบียบบน NotebookLM หลังผ่าน Flow นี้

เมื่อคุณรัน Flow นี้จนจบในแต่ละเลกเชอร์ ไฟล์ที่จะอัปโหลดเข้าไปจัดเก็บในห้องสมุด **NotebookLM** จะมีเพียง 3 ไฟล์หลักนี้เท่านั้นครับ:

1. **`Curriculum_Map.md` (เวอร์ชันล่าสุดเสมอ):** อัปโหลดทับไฟล์เดิม เพื่อให้ NotebookLM มีสารบัญนำทางและแผนผังความเชื่อมโยงของวิชานี้ทั้งหมด
2. **`notes-optimized-XX.md`:** โน้ตเนื้อหาหลักของแต่ละเลกเชอร์ที่ตัดความซ้ำซ้อนข้ามคาบออกแล้ว (ใช้สำหรับสืบค้นอ้างอิงละเอียด)
3. **`notes-crystallized-XX.md`:** โน้ตสรุปตกผลึก High-Yield และแผนปฏิบัติการจริง (ใช้สำหรับสั่งให้ NotebookLM เจนเนอเรตแบบฝึกหัด, ทำ Study Guide หรือสรุปย่อก่อนสอบ)

## 💡 ทำไมการเรียงลำดับแบบนี้ถึงสมเหตุสมผลที่สุดทางคลินิก?

- **ทำไมต้อง Synthesize ก่อน Enrich?** → เพื่อให้เรานำคำพูดขยายความของอาจารย์ในห้องเรียน (Spoken Pearls) มาร่วมรับการตรวจสอบความถูกต้องเชิงวิชาการ (Enrich & Verify) และเขียนกลไกกำกับไปพร้อมกันทีเดียว
- **ทำไมต้อง Enrich ก่อน Resequence?** → เนื้อหาที่เสริมความลึกแล้ว (กลไก, `‼️` flags, callouts) จะถูกจัด cluster ได้ถูก layer มากขึ้น เพราะ resequencer ต้องการเนื้อหาที่สมบูรณ์จึงจะ assign layer ได้แม่นยำ ไม่ควรนำเนื้อหากึ่งสำเร็จมาจัดลำดับก่อน
- **ทำไมต้อง Resequence ก่อน Track?** → curriculum-tracker จะเปรียบเทียบเนื้อหากับ Curriculum_Map ได้มีประสิทธิภาพกว่า เพราะหัวข้อที่ถูก cluster ตาม layer แล้วจะ match กับโครงสร้างของ map ที่จัดตามหลักการแพทย์เช่นกัน ลดโอกาส false-negative ที่เนื้อหาเดิมถูกตีว่าเป็นของใหม่เพราะอยู่คนละบริบทกัน
- **ทำไมต้อง Track ก่อน Crystallize?** → เพื่อตรวจสอบความซ้ำซ้อนข้ามเลกเชอร์ก่อน หากเนื้อหานั้นเคยเรียนแล้วและถูกยุบในขั้น Tracker เราจะได้ไม่เสียเวลาและโควตาโทเค็นไปกับการ "ตกผลึก (Crystallize)" ข้อมูลเดิมซ้ำ ทำให้สรุปใบสุดท้ายมีเฉพาะเรื่องใหม่ที่เป็นประโยชน์จริงๆ ครับ
